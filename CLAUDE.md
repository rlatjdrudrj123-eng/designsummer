# CLAUDE.md · Design Summer 2026 마이크로사이트

K-PRINT 2026 부설 디자인 세미나 "Design Summer"의 홍보용 독립 마이크로사이트.
**모든 UI 작업 전에 `DESIGN_GUIDE.md`를 읽는다. 충돌 시 가이드가 이긴다.**

## 1. 행사 정보 (고정값)

- 행사명: Design Summer (2026°C)
- 일시: 2026.08.20(목) ~ 08.21(금), 2일
- 장소: KINTEX 제2전시장 301호
- 정원: 150명
- 주최: K-PRINT (한국이앤엑스)
- 신청: 사이트 내 폼 없음. K-PRINT 공식 홈페이지로 링크 아웃 (URL은 site.json의 `applyUrl`)
- D1 컨셉: HEAT SOURCE · 열원 · "생각의 온도를 올리는 날"
- D2 컨셉: HEAT TRANSFER · 전사 · "생각이 실물에 닿는 날"

## 2. 스택 (변경 금지)

- Next.js(App Router) + TypeScript
- 스타일: 전역 CSS 토큰 + CSS Modules. **Tailwind·UI 라이브러리·애니메이션 라이브러리 사용 금지** (모션은 CSS와 rAF로 직접)
- 아이소메트릭 씬: Canvas 2D + rAF 직접 구현. 게임엔진·three.js 금지
- 호스팅: Vercel. 이미지 저장: Vercel Blob
- 폰트: Pretendard Variable(CDN), Instrument Sans(구글 폰트). `--font-display` 변수로 추상화

## 3. 라우트

- `/` : 원페이지 (히어로 → 씬 → D1 챕터·라인업 → D2 챕터·라인업 → 타임테이블 → 오시는 길 → 신청 → 푸터)
- `/admin` : 이미지 관리. `ADMIN_PASSWORD` 환경변수로 보호 (미들웨어에서 쿠키 세션)

초기 데이터 (첨부 원고 기준, sessionDesc·credentials는 원고에서 압축):

- D1: 로호타입 김기창(타이포·인쇄) / 오디너리피플 서정민 · "브랜드를 만드는 세 가지 질문" / 우트크리에이티브 고성우 · "필요 없는 건 다 끕니다" (3D 모션) / 아카브릭 조명훈 · "AI 시대, 디자이너는 무엇에 집중해야 하는가?"
- D2: 아우라지디자인 박진택 · "컨셉을 관통하는 창의적인 브랜드 비주얼" / 오세븐 배수규 · "디자인은 취향이 아닌 비즈니스다" / 태창금박지 이호준 · "디자인을 완성하는 박인쇄의 힘" / 패터니스튜디오 이요안나 · "패턴디자인 출력에서 고려해야 할 내용"

크레덴셜은 원고의 사실만 사용, 과장 수식 금지.

## 5. 아이소메트릭 씬 스펙

- 위치: 히어로 다음 풀스크린 섹션. 제목 없이 한 줄 안내만: "301호를 미리 걸어보기" + 조작 안내
- 좌표계: 타일 그리드(권장 14×10), 2:1 다이메트릭 투영. 월드 좌표 ↔ 스크린 좌표 변환 유틸 분리
- 렌더: Canvas 2D. SVG 에셋을 Image로 로드해 깊이 정렬(y 기준) 후 드로우. 디바이스 픽셀 비율 대응
- 에셋: `/public/scene/*.svg`. 전부 코드로 직접 작성 (외부 다운로드 금지). DESIGN_GUIDE.md 10장 아트 디렉션 준수
- 엔티티:
  - 캐릭터 1 (걷기 2프레임)
  - 벽 포스터 8: D1 벽 4장, D2 벽 4장. 면에는 어드민에서 업로드한 포스터 이미지(`poster-{speakerId}`)를 투영해 표시, 미업로드 시 열 색 플레이스홀더
  - D1 군중, D2 군중: 접근 시 각 날의 컨셉 카드 (HEAT SOURCE / HEAT TRANSFER)
  - 안내데스크: 접근 시 신청 안내 카드 + applyUrl 버튼
- 조작: PC는 클릭 지점 이동 + 방향키, 모바일은 탭 지점 이동. 경로는 직선 이동으로 충분 (장애물 회피 불필요한 배치로 설계)
- 트리거: 오브젝트 반경 1.2타일 진입 시 말풍선 프롬프트 → 클릭/탭 또는 Enter로 카드 오버레이(DOM) 열림. Esc·바깥 탭으로 닫힘
- 캐릭터 이동 잔열: 지나간 타일에 1초간 열이 남았다 식음 (가이드 7장 물리)
- 폴백: `prefers-reduced-motion` 또는 캔버스 불가 시, 씬 대신 정적 아이소메트릭 일러스트(SVG) + 포스터·D1·D2·신청으로 가는 버튼 4개. 씬의 정보는 전부 아래 섹션들에도 존재하므로 SEO·접근성 문제 없음

## 6. 어드민 스펙

- 기능은 이미지 관리만: KV 1, 연사 프로필 8, 세션 포스터 8 슬롯. 슬롯별 업로드·교체·삭제
- 업로드 시 webp 변환·리사이즈 (프로필 800px, 포스터 1200px)
- 텍스트 수정은 어드민 범위 밖. speakers.json을 직접 수정 (이 결정을 어드민 화면에 명시)
- 디자인: 시스템 폰트, 흰 바탕, 헤어라인 표. 본 사이트 토큰 미사용

## 7. 빌드 순서 (Phase 통과 전 다음 단계 금지)

1. **토큰·뼈대**: 전역 CSS 토큰(가이드 4·5·7장), 레이아웃 그리드, 폰트 로딩, 빈 섹션 골격
2. **히어로**: 열 블롭 캔버스(호흡 + 커서 열원), 타이포 배치, °C 스크롤 카운터
3. **콘텐츠 섹션**: D1/D2 챕터, 라인업 편집 레이아웃, 타임테이블, 오시는 길, 신청, 푸터
4. **아이소메트릭 씬**: 좌표계·렌더러 → SVG 에셋 → 이동·트리거 → 카드 오버레이 → 폴백
5. **어드민**: 인증, Blob 업로드, 슬롯 UI, 본 사이트와 이미지 연결
6. **폴리시**: 모바일 390px 검수, reduced-motion, 성능(씬 60fps, LCP 2.5s), 가이드 12장 체크리스트 전 항목

## 8. 작업 규칙
