## 0. 빌드 메모 (구현 제약 — 읽고 시작)

- **결정론적 집계.** 런타임 AI 없음. 질문 8개의 선택지마다 동물에 점수를 더하고, **총점 최고 동물 = 결과**. 단순 `reduce` 1방.
- **동점 처리(tie-break):** 동점이면 **온도 낮은 동물 우선**(아래 표의 `tieOrder` 작은 값 = 차가운 쪽). 결정론 보장.
- **🦢 백조 = 히든 결과.** 메인 9종과 별개. **점수가 고르게 분산돼 뚜렷한 1위가 없을 때** 등장(아래 2장 트리거 규칙). 겉은 균형, 속은 발버둥 = 백조 테마와 일치. "나 히든 떴어 ㅋㅋ" → 재시도·공유 유발 이스터에그.
- **스택 정합성:** Next.js 16 / React 19. OG/트위터 메타는 `app/test/page.tsx`의 `generateMetadata({ searchParams })`로, 결과별 OG 이미지는 `next/og`(ImageResponse) 라우트(`?r=` 분기)로 뽑는다. 카피는 전부 이 문서에서 가져온다.
- **브랜드 고정값(임의 변경 금지):**
  - 행사명: `디자인 썸머 일산` / `Design Summer 2026`
  - 슬로건: `the creative heatwave`
  - 날짜: `2026.08.20(목)–08.21(금)` · 장소: `KINTEX 제2전시장`
  - 도메인: `https://design-summer.kr`
  - 비주얼 테마: 사이트가 이미 히트(열·온도) 그래픽(HeatBlob/CursorHeat)을 쓰므로 **온도(°C) 컨셉이 브랜드와 자연 연결됨.** 결과 카드 색 = 온도 스펙트럼(차가운 파랑 25°C → 뜨거운 빨강 120°C).
- **카톡 공유 핵심:** OG 이미지 + OG title/description이 전부다. 텍스트 짧게, 동물 이모지가 썸네일에서 바로 보이게.

## 1. 동물 유형 — 4 대립축 × 9종 (+ 히든 1)

테스트는 **4개의 대립축**으로 디자이너 성향을 가른다. 양극이 각각 하나의 결과 동물.
부엉이 🦉는 협업축에 "**아예 혼자**"라는 제3의 결을 더하는 **축 밖 독립형**(9번째).

| 축 | 한쪽 끝 | 반대쪽 끝 |
|---|---|---|
| **취향** (마이너 ↔ 대중) | 🦙 알파카 | 🦎 카멜레온 |
| **속도** (완벽주의 ↔ 마감버프) | 🦥 나무늘보 | 🦊 여우 |
| **협업** (수용 ↔ 주도 / +독립) | 🐕 강아지 · 🐯 호랑이 | 🦉 부엉이(독립) |
| **동력** (영감 ↔ 시스템) | 🐈 고양이 | 🦦 수달 |

### 커버리지 체크 (밸런스 검증)
9동물 각각 **최소 3개 문항**에서 득점 기회. 최대 점수도 7~9점대로 평탄.

| 동물 | 득점 가능 문항 | 최대 점수(이론치) |
|---|---|---|
| 알파카 🦙 | Q1, Q3, Q4, Q8 | 9 (2+2+2+3) |
| 나무늘보 🦥 | Q2, Q6, Q8 | 7 (2+3+2) |
| 부엉이 🦉 | Q1, Q2, Q5 | 7 (2+2+3) |
| 고양이 🐈 | Q2, Q4, Q6, Q8 | 8 |
| 강아지 🐕 | Q3, Q4, Q5, Q7 | 8 |
| 카멜레온 🦎 | Q1, Q3, Q4 | 7 (2+2+3) |
| 수달 🦦 | Q1, Q5, Q6, Q7 | 8 |
| 여우 🦊 | Q2, Q5, Q7, Q8 | 9 (3+2+2+2) |
| 호랑이 🐯 | Q3, Q6, Q7 | 7 (3+2+2) |

> **알고리즘 의사코드**
> ```
> scores = {alpaca:0, sloth:0, owl:0, cat:0, dog:0, chameleon:0, otter:0, fox:0, tiger:0}
> for each answered choice: scores[animal] += points
>
> sorted = animals sorted by (score desc, tieOrder asc)
> top = sorted[0]; second = sorted[1]
> scoredCount = number of animals with score >= 1
>
> // 🦢 백조 히든: 1위가 뚜렷하지 않고 점수가 넓게 퍼졌을 때
> if (top.score - second.score) <= 1 AND scoredCount >= 6:
>     result = swan
> else:
>     result = top
> ```
> 백조 트리거 임계치(`<=1`, `>=6`)는 출시 후 GA4 분포 보고 1회 튜닝.

## 3. 결과 화면 카피

### 인트로 화면 (테스트 시작)

- **헤드라인:** `당신은 {이모지} {유형명}` 예) `당신은 🦉 헤드셋 낀 은둔형 부엉이`

- 변형 A(짧게): `이거 완전 나잖아 ㅋㅋ 너도 해봐`
- 변형 B(밈): `우리 팀 동물상 다 모으면 사파리 완성된다`
- 백조 전용: `히든 떴다 🦢 너도 백조 나오나 해봐`

### 공유 URL 규칙
- 결과 공유 링크에 동물 식별 쿼리: `https://design-summer.kr/test?r={id}`
  (`id`: alpaca / sloth / owl / cat / dog / chameleon / otter / fox / tiger / swan)
- `generateMetadata`가 `?r=`로 동물별 OG title/description/이미지 매핑. 파라미터 없으면 공통 OG로 폴백.
- `?r=` 진입 시 = 공유받은 사람 → 해당 결과 카드 + "나도 측정하기" 버튼.
