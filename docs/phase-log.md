### heatmap 튜닝 기록(시행착오)

- 투명 배경 'lighter'는 R을 255로 포화시켜(밀도가 alpha에 감) 전면 코어레드 → **불투명 검정 배경**으로 R=진짜 밀도 확보
- 색 변환 후 블러는 옐로 가장자리를 잃음 → **밀도 단계에서 블러**(옐로→오렌지→레드 띠 보존)
- alpha 과다=포화, alpha 과소=창백 → 겹침으로만 core 형성되도록 0.34~0.54 조정. LUT main 구간 앞당겨 KV 채도(레드오렌지 본체) 일치

## Phase 2 보완 · 3차 (2026-06-12) — 열 필드 자동 캘리브레이션

손튜닝 종료. KV 픽셀에 자동 피팅하는 빌드타임 파이프라인으로 전환.

### 파이프라인 (`scripts/calibrate-heat.mjs`, 라이브러리 없음 / `npm run calibrate`)

1. **PNG 디코드/인코드**: Node 내장 zlib 로 직접 구현 (colorType 6, 8bit). kv.png = 3705×5240
2. **열 스칼라**: `heat = (R/255)·(1−G/255)` — 흰=0, 검정(텍스트)=0, 옐로→오렌지→레드로 증가. 박스 다운샘플로 저해상 히트 타깃 생성
3. **색 LUT**: kv.png 의 따뜻한 픽셀(R>140)을 heat 로 빈닝해 밀도→색(흰→옐로→오렌지→레드) LUT 직접 추출. 손으로 색 안 정함
4. **필드 모델**: 큰 가우시안 양수 열원 7 + 흰 만 만드는 음수 소스 3, 좌표에 2옥타브 도메인 워프(작은 진폭). 작은 반경 소스 다수 금지(잔돌기 원인 제거)
5. **피팅**: 고열/저열 픽셀로 소스 초기화 후 힐클라임(14000 iters, 스텝 감쇠, 워프 시드 탐색)으로 MSE(밀도 vs 히트 타깃) 최소화. 결과를 `content/heat-field.json` 에 정지 상태로 저장 (포트레이트 MSE 0.0068, 랜드스케이프 0.0136)

## Phase 2 보완 · 4차 (2026-06-12) — 필드 보완 3건 + °C 정리 (Phase 2 마감)

### 1. LUT 오염 제거

- 추출에서 무채색 제외: 채도 `(max−min)/max < 0.12` 또는 어두운 픽셀(검정 타이포·QR·회색) 배제, 따뜻한 색(R≥G, R>B)만 샘플
- 빈마다 평균 → **중앙값** (outlier 오염 방지)
- 저밀도(0~0.15) 구간 sqrt 빈닝으로 조밀하게 (전체 빈의 ~39% 할당)
- 워름스(warmth) **단조 증가 강제**: 내려가는 빈은 직전 색 hold → 회색 딥 제거
- 렌더: 업스케일 imageSmoothing 활성(기존). 프리뷰는 최근접 업스케일 대신 필드 직접 고해상 렌더로 교체(점 노이즈 원천 제거). 추가 블러 불필요 판단
- **검증: fit 이미지 2장 회색 픽셀 0 / 244,008 (0.000%)** — 채도<0.08 비백색 픽셀 스캔

### 2. 데스크톱 구도 재정의

- 타깃: KV 중앙 띠(구름 구간 18~58%)를 가로 확장(AR 2.0 스트레치)한 크롭, 프레임 상부 62%에 배치 + 하단은 순백 행 (MSE-페널티 충돌 제거)

## Phase 2 보완 · 5차 (2026-06-12) — 가로형 KV 불일치 수정

사용자 지적: 가로형(데스크톱)이 KV 와 안 맞음. 원인: 띠 크롭(y 0.18~0.58) 타깃이 (1) 구름 상단을 잘라 화면 위에 들러붙은 수평 띠가 되고 (2) AR 2.0 가로 스트레치가 KV 의 덤벨(두 로브 + 가는 대각 허리) 구조를 소시지로 뭉갬.

### 시도와 실패 기록

1. **덤벨 전체 크롭 + 세로 압축 2×**: 두 로브가 같은 행으로 병합 → 여전히 수평 띠. 압축 방식의 구조적 한계
2. **로브 기하 합성 타깃 (채택)**: KV 에서 실측한 로브 기하를 가로 구도로 재배치 — 좌상 큰 로브(좌측 잘림, u0.17 v0.34 r0.30) · 가는 허리(r0.12~0.14, s0.4) · 우중단 로브 B(u0.66 v0.48 r0.28) · 우측 어깨 · 희미한 꼬리. 가로 KV 가 없으므로 구도는 합성이되 기하·색·전이는 KV 실측

### 파라미터 수렴

- 순백 페널티 재조정: 합성 타깃이 순백을 내장하므로 가드 수준(minFrac 0.48, weight 10)으로. 과한 가중치는 MSE 와 싸워 옐로 대역을 좁히는 부작용 확인
- 힐클라임 14000→24000 반복
- 최종: **순백 46.0% (≥45% 충족)**, 가장자리 전이 폭 fit med 14px@180 (KV 포트레이트 특성 9~10px@88 과 동급 스케일), 포트레이트 변경 없음

### 렌더 검증 (육안 오판 사건)

캡처가 프리뷰와 달라 보여 데이터/번들/런타임을 격리 — JSON 직접 렌더 vs 페이지 캡처 필드 영역의 밀도 그리드 비교 결과 **상관 0.957, MAE 0.03**: 페이지는 데이터와 일치, 타이틀 오버레이가 육안 비교를 흐린 것. 렌더 버그 아님. (격리 절차: served chunk 에 신규 값 존재 확인 → 런타임 수식 직접 렌더 → 수치 비교)
