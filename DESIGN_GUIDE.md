## 1. 금지 목록 (anti AI-look)

아래 항목은 발견 즉시 수정 대상. "흔한 AI 사이트"로 보이는 즉각적 신호들이다.

- 중앙 정렬 히어로(타이틀·서브카피·CTA 수직 스택) 금지
- 3분할 기능 카드 그리드, 균질한 카드 나열 금지
- 보라·파랑 그라디언트, 글래스모피즘, 네온 글로우 금지
- 모든 섹션에 일괄 적용된 fade-up / slide-up 등장 금지
- hover 시 scale(1.05) + 그림자 커지는 패턴 금지
- border-radius 12~16px의 일괄 적용 금지. 이 사이트의 기본 radius는 0
- Inter, Roboto 등 디폴트 서체 금지
- 이모지, 설명조 카피, "혁신적인" 류 수식어 금지
- 다크모드 변주, 파란·차가운 색 변주 금지 (KV 위반)
- 바랜 크림색 배경(#F4F1EA 계열) 금지. 바탕은 순백

## 3. 컨셉 원리 · 열 시스템

사이트의 모든 시각·모션 결정은 하나의 물리를 따른다: **흰 종이 위에 열이 닿으면 번진다.**

- 바탕은 항상 순백. 열(색)은 국소적으로만 존재하고, 경계는 블러로 번진다
- 인터랙션 = 열의 전달. 마우스·터치·스크롤·캐릭터 접근이 열원이고, 닿은 곳이 데워진다
- D1 = HEAT SOURCE(열원): 관점·질문·기준. 생각의 온도를 올리는 날
- D2 = HEAT TRANSFER(전사): 박·패턴·인쇄·시장. 생각이 실물에 옮겨 붙는 날
- 두 날의 온도 차등은 없다. 열이 생기는 곳과 옮겨가는 곳, 방향만 다르다

## 4. 컬러 토큰 (KV 실측값)

```css
:root {
  --white: #FFFFFF;        /* 바탕. 화면의 70% 이상 */
  --ink: #0A0A0A;          /* 타이포, 씬 라인워크 */
  --heat-core: #C43D1C;    /* 가장 뜨거운 중심. 좁게 */
  --heat-main: #E84A20;    /* 주 레드오렌지. 열 덩어리의 본체 */
  --heat-mid:  #ED7D1D;    /* 오렌지. core/main과 edge 사이 */
  --heat-edge: #F2A100;    /* 옐로. 열의 가장자리, 번짐의 끝 */
}
```

사용 규칙:

- 열 색은 단색 면으로 쓰지 않는다. 반드시 core→main→mid→edge→white로 번지는 radial 그라데이션 또는 blur 스택으로만 등장
- 예외: 씬 내부의 작은 포인트(포스터 면, heat ring), 텍스트 하이라이트는 --heat-main 단색 허용
- 텍스트는 --ink 단일. 열 색 위에 텍스트를 얹을 때도 --ink (KV와 동일)
- 회색 계열 추가 금지. 위계는 회색이 아니라 크기·굵기·여백으로

## 5. 타이포그래피

- 한글: **Pretendard Variable**
- 영문·숫자 디스플레이: **Instrument Sans** (KV의 헬베티카 계열 인상에 가장 가까운 무료 가변 서체. 사내 Helvetica 라이선스 확보 시 교체 가능하도록 font-family 변수화)
- 서체는 이 둘로 끝. 세리프·모노 추가 금지

스케일 (clamp 기반, px 환산 기준):

```css
--type-hero: clamp(56px, 11vw, 160px);   /* Design Summer, D1/D2 챕터 타이틀 */
--type-h2:   clamp(36px, 6vw, 88px);     /* 스튜디오명, 섹션 헤드 */
--type-h3:   clamp(22px, 3vw, 40px);     /* 세션 제목 */
--type-body: 17px;                        /* 본문, 줄간 1.7 */
--type-meta: 13px;                        /* 날짜·장소·°C 등 라벨, letter-spacing 0.04em */
```

규칙:

- 디스플레이 타이포는 화면 밖으로 잘려도 된다 (OFFF 원칙). 단 정보성 텍스트는 잘리지 않는다
- 세로쓰기: 연사 스튜디오명 리스트는 KV처럼 `writing-mode: vertical-rl` 세로 배치를 시그니처로 사용. 모바일에서도 유지
- 숫자 모티프(08.20 / 2026°C / 301)는 meta 스케일로 화면 모서리에 배치. KV의 모서리 날짜 배치를 상속
- 자간: 한글 디스플레이 -0.02em, 본문 0

## 6. 그리드 · 여백 리듬

- 데스크톱 12컬럼, 거터 24px, 좌우 마진 max(40px, 4vw). 모바일 4컬럼, 마진 20px
- 정렬 기본은 비대칭. KV처럼 좌상·우상·우중·좌하에 요소를 흩어 배치하고, 큰 여백을 의도적으로 남긴다
- 섹션 호흡은 균질하지 않게. 밀도 곡선: 히어로(광활) → 씬(꽉 참) → D1/D2 챕터(광활) → 라인업(리듬) → 타임테이블(조밀) → 신청·푸터(광활)
- 섹션 사이 패딩은 120px 디폴트가 아니라 위 곡선에 맞춰 개별 결정

## 7. 모션 원칙

모든 움직임은 열의 물리를 따른다. 장식 모션은 없다.

```css
--ease-heat: cubic-bezier(0.16, 1, 0.3, 1);
--dur-touch: 240ms;   /* hover, 포커스 반응 */
--dur-bloom: 600ms;   /* 열 번짐, 카드 등장 */
--dur-amb:   3~6s;    /* 히어로 블롭 등 환경 모션 */
```

- 등장 모션: fade-up 전면 금지. 대신 "열 번짐 등장": opacity 0→1 + blur(12px)→0, 위치 이동 없음. 종이에 열이 배어 나오는 인상
- 마우스 = 열원: 히어로와 라인업에서 커서 주변에 열 번짐(radial gradient follow, 캔버스 또는 CSS mask). 커서가 지나간 자리는 1초간 잔열이 남았다 식는다
- hover: scale 금지. 텍스트 뒤에서 열이 배어 나오거나(--heat-main blur), 밑줄이 열로 그어지는 방식
- 스크롤: 섹션 reveal이 아니라 상태 변화와 연동. 예: D2 진입 시 열 블롭의 위치가 이동
- marquee는 한 곳만 허용 (신청 섹션 위 "find your heatmap" 라인). 두 곳 이상 금지
- `prefers-reduced-motion`: 환경 모션·잔열·marquee 정지, 번짐 등장은 단순 opacity로 대체

## 8. 섹션별 규정

### 히어로
- KV를 이미지로 박지 않는다. 열 블롭을 캔버스로 재현해 살아 있게 (느리게 호흡 + 커서 열원 반응)
- 타이틀 "Design Summer"는 KV처럼 우측 치우침, "2026°C" 위첨자, 좌상단 08.20 · 우상단 08.21
- 하단에 연사 스튜디오명 8개 세로쓰기 리스트 (KV 레이아웃 상속). 각 이름이 라인업 섹션 앵커
- CTA 버튼을 히어로에 두지 않는다. 스크롤 유도 장치를 따로 두지 않는다 — 블롭 흐름과 콘텐츠로 충분

### 아이소메트릭 씬 (10장 아트 디렉션 참조)
- 도입 문구 한 줄 + 조작 안내(키보드·탭)만. 씬 자체가 설명이 되게
- 씬에서 열람한 포스터·설명 카드는 DOM 오버레이로, 씬 위에 종이가 올라오는 인상

### D1 / D2 챕터
- 각 날의 시작은 챕터 페이지처럼: HEAT SOURCE / HEAT TRANSFER 대형 타이포 + 한 줄 정의 + 날짜
- D1 한 줄: "생각의 온도를 올리는 날" · D2 한 줄: "생각이 실물에 닿는 날"

### 라인업
- 연사 카드 그리드 금지. 한 명 = 한 단락의 편집물: 스튜디오명(h2, 일부 세로쓰기 혼용) + 세션 제목(h3) + 핵심 크레덴셜 2~3줄
- 8명이 같은 레이아웃을 반복하지 않는다. 좌·우 정렬을 교차하고 여백 리듬을 다르게
- hover 시 해당 연사 영역에 열 번짐 + 프로필 사진이 잔상처럼 떠오름 (플로팅)

### 타임테이블
- 표 형태, 헤어라인(1px --ink) 사용. radius 없음. D1/D2 탭 전환
- 현재 진행 중 세션 표시는 행 좌측의 작은 heat dot 하나로

### 신청 · 푸터
- 신청은 K-PRINT 공식 홈페이지로 링크 아웃. 버튼은 --ink 바탕 + --white 텍스트의 직사각형, radius 0
- 푸터: K·print 로고, 주최, 문의. 장식 없음

## 9. 카피 톤

- 헤드라인·라벨은 마침표 없이 명사나 구로 종결
- 번역투 금지: "한 장의", "여덟 개의 세션", "behind ~" 류 구문 사용 금지
- 설명조·권유조 최소화. 사실과 정보를 짧게
- 영문은 KV에 있는 어휘 계열만: Design Summer, heatmap, heat source, heat transfer, °C

## 10. 아이소메트릭 씬 아트 디렉션

- 투영: 2:1 다이메트릭 (가로 2px당 세로 1px). 모든 SVG 에셋이 동일 투영 준수
- 공간: KINTEX 301호를 단순화한 흰 방. 바닥은 흰 타일 + 헤어라인 그리드, 벽 2면
- 라인워크: --ink, 시각 두께 1.5px 고정 (줌 레벨과 무관하게 유지)
- 색: 방·집기·캐릭터는 흰 면 + 검정 라인이 기본. 열 색은 포인트에만: 벽 포스터 면, 천장 조명의 빛 웅덩이, D1·D2 군중 위 heat ring, 캐릭터 접근 시 바닥에 번지는 잔열
- 오브젝트 면에 그라디언트 금지. 열 표현(빛 웅덩이·잔열·ring)만 radial 그라데이션 허용
- 캐릭터: 2등신 단순 실루엣, 얼굴 디테일 없음, 흰 몸 + 검정 라인. 걷기 모션은 2프레임 교차로 충분
- 그림자: 모든 오브젝트 공통, --ink 8% 단일 톤, 같은 방향
- 엔티티: 벽 포스터 8장(D1 벽 4 + D2 벽 4), D1 군중·D2 군중, 안내데스크(신청 링크), 캐릭터 1
- 근접 트리거: 반경 진입 시 오브젝트 위에 말풍선 프롬프트(--white 면 + --ink 라인, radius 0) → 확인 시 카드 오버레이

## 11. 어드민 화면 톤

- 어드민은 꾸미지 않는다. 시스템 폰트, 흰 바탕, 헤어라인 표. 본 사이트 토큰을 가져다 쓰지 않는다

## 12. 셀프 체크리스트 (작업 후 매번)

1. 1장 금지 목록 위반이 한 건도 없는가
2. 화면의 70% 이상이 순백인가
3. 열 색이 단색 면으로 등장한 곳은 없는가 (허용 예외 제외)
4. fade-up·scale hover가 어디에도 없는가
5. 섹션 두 개를 캡처해 나란히 놓았을 때 호흡이 서로 다른가
6. 타이포만으로 어느 행사인지 식별되는가 (KV와 같은 인상인가)
7. 세로쓰기 시그니처가 살아 있는가
8. 모바일(390px)에서 씬·세로쓰기·타임테이블이 모두 동작하는가
9. prefers-reduced-motion에서 정보 손실이 없는가
10. 카피에 번역투·이모지·설명조가 없는가
